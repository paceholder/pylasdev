Для выполнения автоматической увязки кривых Есть такие необходимости:

Нужно сделать так, чтобы прайм мог производить автоматическую увязку кривых, 
по принципу простого сдвига и с возможностью резинки.

По решению этого вопроса есть некоторые идеи:

1) Перед началом обработки программа уточняет с каким разрешением по глубине 
нужно получить результат, если количества данных в кривой достаточно, 
то программа выполняет пункт 2), если данных по глубине не достаточно для 
корректной увязки резинкой, то программа добавляет данные в кривую/кривые 
(уменьшает шаг по глубине) с помощью нелинейной интерполяции. 
Условие достаточности данных может быть например таким: 
Если нужна правильная кривая с шагом 0.1м то программа интерпалировав 
недостающие данные ведёт обработку с шагом 0.01м. 
(Я именно так сейчас и привязываю данные).

2) Для получения относительно соразмерных кривых по всем увязываемым методам, 
вычислять по всем этим методам псевдо пористости.

3) Указываем опорную кривую/кривые, далее программа приняв эту кривую/среднюю 
среди этих кривых привязывает остальные кривые к опорной 
(используя ф-ии взаимо/авто корреляции).

4) При этом реперы привязки должны быть сохранены для каждой из привязываемых 
кривых.

5) Нужна при этом возможность управлять степенью подобия результирующих 
кривых опорным. (Говоря о степени подобия имею ввиду возможность 
управлять "подробностью" увязки экстремумов резинкой, так например при работе 
программы в режиме увязки простым сдвигом степень подобия будет ~0, 
а при увязке "подробной резинкой" степень подобия будет ~100%.)
